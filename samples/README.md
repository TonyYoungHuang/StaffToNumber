# Module 5 Samples

This folder stores repeatable evaluation inputs for the Module 5 worker.

## Categories

- `samples/clean/`: clean PDFs that are expected to become `final`
- `samples/draft/`: usable but uncertain PDFs that are expected to stay `draft`
- `samples/fail/`: unusable PDFs that must not become a confident `final`

## Files

- `samples/manifest.json`: sample definitions and expected outcomes
- `samples/reports/latest.json`: latest structured evaluation result
- `samples/reports/latest.md`: latest human-readable summary
- `samples/reports/history/`: timestamped history snapshots generated by the evaluator

## Usage

From the repo root:

```bash
npm run build
npm run evaluate:samples
```

Each run compares against `samples/reports/latest.json` and records:

- `final` / `draft` / `failed`
- preview hash changes
- expectation mismatches
- category summary
