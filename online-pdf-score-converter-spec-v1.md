# 在线 PDF 乐谱互转工具 Spec v1

## 1. Problem Statement

### 1.1 产品要解决的问题
当前市面上的乐谱处理工具，对普通音乐学习者并不友好。

主要问题包括：

- 现有工具往往过于专业、复杂，普通用户难以上手
- 用户手里的常见输入并不是结构化乐谱文件，而是 PDF 乐谱
- 用户希望把现有乐谱快速转换成另一种记谱形式，用于直接练习或教学
- 手工重录乐谱成本高、耗时长，尤其对个人学习者来说门槛过高

### 1.2 目标用户
本产品一期主要服务：

- 普通乐器学习者
- 个人音乐爱好者

这些用户的典型需求是：

- 将五线谱 PDF 转成可直接练习的简谱
- 将简谱 PDF 转成五线谱，用于教学、整理或打印

### 1.3 核心用户痛点
本项目优先解决的不是“做一个专业制谱软件”，而是：

- 让普通用户用更低门槛的方式完成 PDF 乐谱互转
- 减少现有工具过于复杂、过于专业带来的使用障碍
- 尽量避免用户从头手工录入整份乐谱

### 1.4 一期的优先问题排序
一期按以下优先级解决问题：

1. 五线谱 PDF -> 简谱
2. 简谱 PDF -> 五线谱

其中第一项是主线能力，第二项是一期必须具备但优先级略低的能力。

---

## 2. Proposed Solution

### 2.1 产品定位
本产品定位为：

**一款在线 PDF 乐谱互转工具**

它不是大型 SaaS，也不是多端复杂系统。一期重点是把“上传 PDF -> 转换 -> 下载结果”这条链路做成可用产品。

### 2.2 一期产品形态
一期产品采用 Web 形态，面向 PC 使用场景。

整体由两部分组成：

1. **SEO 官网**
   - 用于被 Google 搜索到
   - 用于介绍产品、能力、使用方式
   - 用于承接海外搜索流量

2. **受权限保护的应用站**
   - 用户登录后上传 PDF
   - 发起转换
   - 获取结果
   - 下载成品或初稿包

### 2.3 一期支持的核心能力

#### A. 五线谱 PDF -> 简谱
- 面向清晰、规整的五线谱 PDF
- 目标是生成可直接打印或使用的简谱 PDF
- 这是一期最核心的主能力

#### B. 简谱 PDF -> 五线谱
- 面向常见简谱排版
- 目标是生成可直接打印或使用的五线谱 PDF
- 复杂记号和特殊格式一期不保证

### 2.4 一期的结果交付策略
系统对用户上传的 PDF 进行识别和转换后，分两类结果交付：

#### 情况 1：识别质量较高
直接交付成品文件：

- 五线谱 PDF -> 简谱 PDF
- 简谱 PDF -> 五线谱 PDF

#### 情况 2：识别质量不足
交付“初稿包”，用于后续人工修正：

- 一份可阅读的 PDF 结果
- 一份可修改的结构化文本

系统需要明确告诉用户：

**这是初稿，需人工修正。**

### 2.5 用户体验策略
前台体验应尽量像在线即时工具：

- 用户上传文件
- 短时间内尽量直接返回结果
- 如果处理时间过长，则平滑切换为“处理中，稍后查看”

也就是说，一期前台以“即时返回体验”为目标，后台允许必要时异步处理。

### 2.6 商业与使用路径

#### 中国大陆路径
- 产品在国内电商平台售卖
- 用户购买后获得专属激活码
- 用户注册账号并兑换激活码
- 激活后获得 1 年使用权限
- 中国大陆用户不走站内支付

#### 海外路径
- 用户通过 Google 搜索进入官网
- 可先获得有限次数试用
- 后续可接海外支付接口

### 2.7 权限模型
一期采用较稳妥的账号制：

- 登录方式：邮箱 + 密码
- 激活码先兑换为账号权限
- 账号权限有效期：1 年

### 2.8 输出形式优先级
一期的输出优先级如下：

1. 可直接下载并打印的 PDF 成品
2. 当质量不足时，下载可人工修正的初稿包
3. 网页预览为辅助能力，不是核心交付物

## 3. Technical Constraints

### 3.1 一期的技术实现原则
一期优先目标是“尽快做出可用产品”，而不是一开始就自研全套识别能力。

因此技术策略为：

- 优先使用现成能力组合完成 PDF 识别、OCR、解析与转换
- 通过自定义规则、后处理和结果分级提升可用性
- 在后续版本中再逐步替换关键环节

### 3.2 输入约束
一期主要输入为 PDF：

- 五线谱 PDF
- 简谱 PDF

重点支持：

- 清晰
- 排版规整
- 单旋律为主
- 少量简单伴奏尽量尝试

### 3.3 支持范围约束

#### 五线谱 PDF -> 简谱
优先支持：

- 清晰的单旋律谱
- 排版规整的常见乐谱 PDF
- 可作为练习用途的结果输出

#### 简谱 PDF -> 五线谱
优先支持：

- 常见简谱排版
- 基础节奏和常见记号

暂不保证：

- 特殊记号密集的谱例
- 高复杂度排版

### 3.4 复杂度控制策略
一期不强调僵硬的页数规则，而更强调识别复杂度控制：

- 允许用户上传 PDF
- 前端只做基础文件大小限制
- 后端根据清晰度、复杂度、排版情况决定结果质量分级
- 尽量少做“上传前拒绝”，更多采用“转换后分级交付”

### 3.5 结果分级机制
系统应具备简单、稳定的结果分级：

- 可直接使用的成品
- 需人工修正的初稿

该分级机制应尽量少给用户“被拒绝感”，而是以结果导向为主。

### 3.6 市场与语言约束
一期优先面向中文市场：

- 中文用户体验优先
- 官网需要具备 Google 可搜索能力
- 海外市场一期主要以 SEO 曝光和后续扩展准备为主

### 3.7 支付与授权约束

#### 国内
- 不做站内支付
- 通过电商平台售卖
- 通过激活码发放使用权限

#### 海外
- 可预留或接入站内支付接口
- 免费试用按次数限制

## 4. Non-goals

以下内容明确不属于一期目标。

### 4.1 不做网页内编辑
一期不做网页内乐谱编辑，也不做可视化修谱。

网页只负责：

- 上传文件
- 发起转换
- 返回结果
- 下载成品或初稿包

编辑能力后置到后续版本。

### 4.2 不承诺复杂谱例的高质量支持
一期明确不以复杂谱为目标，包括但不限于：

- 多声部复杂谱
- 钢琴双手复杂谱
- 合唱总谱
- 和声密集谱
- 特殊排版或特殊记号很多的谱例

### 4.3 不做中国大陆站内支付
国内交易不在系统内完成，一期不做中国大陆站内支付能力。

### 4.4 不做多端体系
一期不做：

- 小程序
- 桌面客户端
- 多端同步体系

### 4.5 不做大型 SaaS 能力
一期不做：

- 协作
- 分享社区
- 曲库平台
- 复杂会员体系
- 高级后台运营系统

### 4.6 不追求全谱种、全格式、全场景覆盖
一期不是“万能乐谱识别平台”，而是一个聚焦 PDF 乐谱互转的在线工具。

## 5. Success Criteria

### 5.1 一期最核心成功标准
如果要用一句话判断一期是否成功，标准是：

**大部分清晰、规整、单旋律 PDF 能被成功转换成可直接打印使用的结果。**

### 5.2 主方向成功标准
在五线谱 PDF -> 简谱方向上：

- 大部分目标谱例能够生成可直接使用的简谱 PDF
- 用户不需要重新从头手工录谱
- 结果足以用于个人练习和基础教学

### 5.3 次方向成功标准
在简谱 PDF -> 五线谱方向上：

- 常见简谱排版能产出可用结果
- 对复杂情况至少能给出有价值的初稿包
- 该能力在一期达到“基础可用”

### 5.4 产品体验成功标准
- 用户能完成从上传到下载的完整流程
- 系统在结果不足时能明确提示“这是初稿，需人工修正”
- 用户不会因结果等级不清晰而误解系统能力

### 5.5 商业路径成功标准
- 国内用户可通过电商平台购买后激活使用
- 激活码、账号、权限流程可稳定工作
- Google 可搜索官网能承接基础搜索流量

### 5.6 一期完成的判断标准
以下条件同时满足，可认为一期达到可上线水平：

1. 用户可以注册并登录账号
2. 用户可以兑换激活码并获得 1 年权限
3. 用户可以上传五线谱 PDF 或简谱 PDF
4. 系统可以返回成品或初稿包
5. 用户可以下载结果文件
6. 五线谱 PDF -> 简谱达到主线可用水平
7. 简谱 PDF -> 五线谱达到基础可用水平

## 6. 附加摘要

### 6.1 一期产品摘要
一期不是做一个复杂的乐谱平台，而是做一个：

**面向普通学习者、以 PDF 乐谱互转为核心、支持购买后激活使用的在线工具。**

### 6.2 一期主线能力摘要
- 主线：五线谱 PDF -> 简谱
- 次线：简谱 PDF -> 五线谱
- 结果高质量时交付成品 PDF
- 结果不足时交付初稿包

### 6.3 一期边界摘要
- 不做网页内编辑
- 不做复杂谱高质量承诺
- 不做国内站内支付
- 不做小程序和桌面端
- 不做大而全平台
