## Module 5D implementation summary

### 1. Refined notehead core extraction

Added `refineNoteheadCore(...)` to search for a denser sub-region inside each connected component. This reduces the chance that a stem, accidental, or adjacent symbol dominates the notehead bounding box.

The worker now stores stronger per-candidate geometry:

- `roundness`
- `coreWidth`
- `coreHeight`
- `coreArea`

### 2. Stronger candidate filtering

Candidate selection now uses:

- original component aspect ratio
- refined core aspect ratio
- refined core roundness
- confidence composed from raw shape, density, candidate-column alignment, and refined core quality

This makes candidate acceptance closer to "looks like a notehead" rather than "is some compact dark component".

### 3. Barline / symbol-noise suppression

Added `hasVerticalBarrier(...)` to reduce accidental false positives caused by very tall narrow vertical structures, which often behave more like barlines or non-note noise than true accidentals.

### 4. Structured promotion scoring

Added:

- `estimateSpacingConfidence(...)`
- `computePromotionScore(...)`

Final promotion is no longer based only on note count + average confidence + stem count. It now also considers:

- certainty count
- spacing confidence
- structural note quality
- a single `promotionScore`

### 5. Richer preview output

Final and draft preview text now includes:

- promotion score
- spacing confidence
- certain notehead count

This makes debugging easier when a sample stays in draft.
